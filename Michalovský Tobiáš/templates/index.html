Co je DinnoBird?

🎮

Gameplay

DinnoBird je 2D arcade hra inspirovaná klasickým Flappy Bird. Ovládáš dinosauřího ptáka, který musí proletět mezerami mezi kamennými pilíři v džungli. Každým průletem získáváš bod — jak daleko se dostaneš?

🏆

Soutěžení

Výsledky jsou uloženy do sdílené databáze. Po každé hře se tvoje nejlepší skóre automaticky zapíše a zobrazí v globálním žebříčku. Registruj se a přidej se do souboje o první místo!

🌿

Grafika a zvuk

Hra používá vlastní sprite ptáka, džunglové pozadí a kamenné pilíře. Doprovází ji atmosférická melodie od Sigur Rós, která dotváří tajemnou náladu pralesního dobrodružství.

⚡

Obtížnost

Hra se postupně zrychluje — každý průlet přidává k rychlosti pilířů. Čím vyšší skóre, tím menší prostor pro chybu. Reflexy jsou klíčem k přežití v DinnoBird!

🔗

Web integrace

Hra a webová stránka sdílejí jednu SQLite databázi. Z menu ve hře lze přímo otevřít web, registrovat se, přihlásit — a teprve pak hrát. Web zobrazuje živý žebříček i výsledky z minulých sezení.

🛠️

Technologie

Hra je napsána v Python + Pygame. Backend webu běží na Flask. Data jsou persistována v SQLite. Frontend využívá čistý HTML/CSS/JS bez externích frameworků.

Úvod a cíl projektu

DinnoBird je školní projekt, jehož cílem bylo vytvořit plnohodnotnou 2D arcade hru s persistencí dat a webovým rozhraním. Inspirací byl kultovní Flappy Bird, avšak s vlastní tematikou džungle a dinosaurů.

Požadavky projektu

  • Funkční 2D hra s fyzikou (gravitace, kolize)
  • Ukládání a zobrazování skóre v databázi
  • Systém přihlášení a registrace uživatelů
  • Webový žebříček propojený s hrou
  • Vlastní grafické prostředky (sprity, pozadí)
  • Hudební doprovod

Postup vývoje

Vývoj probíhal ve třech fázích:

  1. Fáze 1 – Herní jádro: Implementace fyziky ptáka, generování pilířů, detekce kolizí a herní smyčky v Pygame.
  2. Fáze 2 – Databáze a autentizace: Návrh SQLite schématu, implementace hash hesel (SHA-256), propojení hry s DB přes modul db.py.
  3. Fáze 3 – Webový server: Vytvoření Flask aplikace s routami pro přihlášení, registraci, žebříček a propojení s databází hry.

Autor

Projekt vytvořil Toby jako součást výuky programování. Veškerý kód je vlastní tvorba s výjimkou herního enginu Pygame a webového frameworku Flask.

Architektura projektu

Projekt se skládá ze tří vzájemně propojených částí, které sdílejí jednu SQLite databázi:

dinnobird/ ├── HororBird.py # Hlavní soubor hry (Pygame) ├── db.py # Databázový modul (sdílený) ├── app.py # Flask webový server ├── dinnobird.db # SQLite databáze ├── templates/ │ └── index.html # HTML šablona webu ├── static/ │ ├── diagram1.png │ └── diagram2.png └── imgs/ ├── Bird.png ├── Džungle.png ├── Pilíř2.png ├── MenuTitle.png └── Konec.png

Komunikace mezi komponentami

Hra (HororBird.py) a Flask server (app.py) oba importují modul db.py, který poskytuje funkce pro přístup k databázi. Tím je zaručena konzistence dat — skóre uložené hrou je ihned viditelné na webu.

Spuštění projektu

# 1. Spustit webový server python app.py # 2. V druhém terminálu spustit hru python HororBird.py # Web server automaticky otevře prohlížeč na: # http://127.0.0.1:5000

Hra při startu menu zkontroluje, zda Flask server běží. Tlačítko „Website" v menu hry otevře web pouze tehdy, pokud server odpovídá na 127.0.0.1:5000.

Herní smyčka a fyzika

Třída Bird

Pták je reprezentován třídou s pozicí (x, y) a vertikální rychlostí (vy). Gravitace se přičítá každý frame, mávnutí nastaví zápornou rychlost (skok nahoru):

GRAVITY = 0.45 # Gravitační zrychlení FLAP_STRENGTH = -9.5 # Rychlost při mávnutí # Každý frame: bird.vy += GRAVITY bird.y += bird.vy # Při stisku SPACE: bird.vy = FLAP_STRENGTH

Generování pilířů

Pilíře jsou generovány procedurálně. Výška horního pilíře se mění o náhodnou hodnotu v rozsahu ±MAX_HEIGHT_DIFF oproti předchozímu, čímž vzniká plynulý, ale nepředvídatelný průjezd:

MAX_HEIGHT_DIFF = 120 # Max. změna výšky MIN_GAP_RATIO = 0.25 # Min. mezera (25% výšky obrazovky) BASE_SPEED = 3.4 # Základní rychlost SPEED_INCREMENT = 0.18 # Zrychlení za každý bod

Detekce kolizí

Hra používá pygame Rect kolize — po každém frame se zkontroluje, zda obdélník ptáka protíná obdélník libovolného pilíře nebo hranice obrazovky. Pokud ano, nastane „game over".

Bodování

Bod se přičte, jakmile střed pilíře přejde za střed ptáka. Po každém bodu se speed zvýší o SPEED_INCREMENT, čímž hra postupně zrychluje.

Countdown

Před každou hrou (i po restartu) proběhne 4sekundový countdown (3, 2, 1, GO!). Během odpočtu pilíře i pták jsou vykresleny, ale fyzika ještě neprobíhá — hráč má čas se připravit.

Databáze a autentizace

Schéma databáze

-- Tabulka uživatelů CREATE TABLE users ( id INTEGER PRIMARY KEY AUTOINCREMENT, username TEXT UNIQUE NOT NULL, password_hash TEXT NOT NULL ); -- Tabulka skóre (jedno nejlepší skóre na hráče) CREATE TABLE scores ( id INTEGER PRIMARY KEY AUTOINCREMENT, player TEXT NOT NULL UNIQUE, score INTEGER NOT NULL ); -- Unikátní index pro rychlé vyhledávání CREATE UNIQUE INDEX idx_scores_player ON scores (player);

Hashování hesel

Hesla nejsou ukládána v čitelné podobě. Při registraci i přihlášení se použije SHA-256 hash:

import hashlib def hash_password(password): return hashlib.sha256( password.encode('utf-8') ).hexdigest()

Ukládání skóre

Po každém konci hry se zkontroluje, zda hráč již má záznam. Pokud ano, skóre se aktualizuje pouze tehdy, je-li nové vyšší než stávající:

existing = db.execute( "SELECT score FROM scores WHERE player = ?", (player,) ).fetchone() if existing is None: db.execute("INSERT INTO scores ...", ...) elif new_score > existing[0]: db.execute("UPDATE scores SET score = ? ...", ...)

Propojení hry s DB

Hra při spuštění volá init_db() z modulu db.py, který vytvoří tabulky (pokud neexistují) a odstraní případné duplicity. Funkce is_registered_user() slouží jako ochrana — do hry se dostane pouze hráč, který existuje v databázi (tj. zaregistroval se přes web).

Webový server (Flask)

Routy

Flask aplikace (app.py) implementuje tyto HTTP endpointy:

GET / → Hlavní stránka (žebříček, login/registrace) POST /auth → Přihlášení nebo registrace uživatele GET /logout → Odhlášení (smazání session) GET /scores.json → API endpoint – vrací JSON se skóre

Správa session

Po úspěšném přihlášení se uloží jméno uživatele do Flask session. Session je šifrována tajným klíčem (secret_key) uloženým v paměti serveru:

app.secret_key = 'supersecret_localhost_1234' # ⚠ Pro produkci použij náhodný bezpečný klíč! # Přihlášení: session['username'] = username # Odhlášení: session.pop('username', None)

Renderování šablony

Flask předává šabloně seznam skóre (načtený z DB), přihlášené uživatelské jméno a případnou chybovou zprávu. Jinja2 šablona pak tyto data dynamicky zobrazí:

return render_template("index.html", scores=get_scores(), username=session.get('username'), error=request.args.get('error') )

JSON API

Endpoint /scores.json vrací aktuální skóre ve formátu JSON — lze jej využít pro budoucí AJAX aktualizace žebříčku bez nutnosti načítat celou stránku:

// Příklad odpovědi: [ {"player": "honzik", "score": 100}, {"player": "hroch", "score": 70}, {"player": "toby", "score": 62} ]

Automatické otevření prohlížeče

Při spuštění app.py se automaticky otevře prohlížeč díky modulu webbrowser — uživatel nemusí ručně zadávat URL do adresního řádku.

Použité technologie

Python 3

Celý projekt je napsán v Pythonu 3. Využívá standardní knihovny jako hashlib, sqlite3, webbrowser, urllib a socket.

Pygame

Herní engine pro 2D grafiku, zpracování vstupů a zvuk. Pygame umožňuje vykreslování spriteů, detekci kolizí pomocí Rect a přehrávání MP3 hudby přes pygame.mixer.

pip install pygame

Flask

Lehký Python webový framework. Zpracovává HTTP požadavky, renderuje HTML šablony pomocí Jinja2 a spravuje uživatelské session.

pip install flask

SQLite

Vestavěná databáze v Pythonu — nevyžaduje žádnou instalaci. Ideální pro lokální projekty a prototypy. Databázový soubor dinnobird.db je sdílen mezi hrou i webem.

HTML / CSS / JavaScript

Frontend webu používá čisté HTML5, CSS3 (custom properties, grid, flexbox, animace) a vanilla JavaScript. Žádné frameworky — vše vlastní kód.

Jinja2

Šablonovací systém integrovaný ve Flasku. Umožňuje vkládat Python data (jako seznam skóre nebo jméno uživatele) přímo do HTML pomocí {{ }} a {% %} syntaxe.
